# 无高低差关卡设计指南

## 概述

本指南专注于日式SRPG（战棋游戏）中**无高低差地形系统**的关卡设计方案。通过深入分析火焰纹章GBA、天地劫等经典作品，提炼出一套完整的替代方案设计框架。

---

## 1. 为什么无高低差？设计取舍分析

### 1.1 高低差系统的优缺点

**高低差优势：**
- 直观的空间策略感
- 天然的高低位置控制权
- 远程单位的战术价值提升
- 符合人类对地形的直觉认知

**高低差劣势：**
- 实现复杂度较高
- AI寻路计算量增加
- 地图设计约束更多
- 对移动端性能要求更高

### 1.2 无高低差的设计价值

**适合选择无高低差的场景：**

| 场景 | 说明 |
|------|------|
| 移动端优先 | 简化计算，提升性能 |
| 快节奏战斗 | 减少地形计算，聚焦单位对抗 |
| 2D像素风格 | 高低差视觉表现受限 |
| 强调兵种克制 | 让克制关系成为核心 |
| 简化学习成本 | 降低新手入门门槛 |

### 1.3 设计核心问题

> **关键问题**：没有高低差，如何创造空间策略深度？

答案：**用地形效果系统替代高度差，用障碍物创造阻塞点，用机制丰富战术选择。**

## 2. 替代高低差的核心机制

### 2.1 地形效果系统（核心替代方案）

地形效果是无高低差系统最核心的替代机制。通过给不同地形赋予独特的效果，创造战术价值差异。

#### 2.1.1 地形效果分类

```yaml
防御地形:
  森林: 
    回避加成: +20%
    移动力消耗: +1
    视觉表现: 树木覆盖
  城墙/堡垒:
    防御加成: +30%
    移动力消耗: +2
    特殊效果: 不可被穿越（仅飞行单位可通过）
  建筑内部:
    防御加成: +15%
    回避加成: +10%
    特殊效果: 远程单位射程-1

移动障碍:
  河流:
    移动力消耗: +3
    职业限制: 仅游泳/飞行单位可停留
  沙漠:
    移动力消耗: +2
    特殊设计: 隐藏路径/宝藏奖励
  雪地/沼泽:
    移动力消耗: +1~2
    效果叠加: 可能附加减速状态

危险地形:
  毒沼:
    持续伤害: 每回合损失10%HP
    战术用途: 驱赶敌人/创造陷阱
  岩浆/火焰:
    即时伤害: 进入时损失20%HP
    阻挡效果: 不可停留
  雷云:
    随机伤害: 20%概率被雷击中

效果地形:
  治疗之泉:
    回复效果: 每回合回复15%HP
  增益石碑:
    属性提升: 攻击力+20%，持续2回合
  传送阵:
    特殊机制: 可传送至指定位置
```

#### 2.1.2 火焰纹章GBA模式（经典参考）

火焰纹章GBA三部曲（封印之剑、烈火之剑、圣魔之光石）是无高低差设计的典范：

**核心地形效果表：**

| 地形 | 回避加成 | 防御加成 | 移动力消耗 | 特殊效果 |
|------|----------|----------|------------|----------|
| 平原 | 0% | 0 | 1 | 无 |
| 森林 | +20% | +1 | 2 | 无 |
| 深山 | +30% | +2 | 3 | 无 |
| 堡垒 | +20% | +2 | 2 | 每回合回复HP |
| 城墙 | 0% | +3 | - | 不可穿越 |
| 河流 | 0% | 0 | 3 | 仅飞行单位可停留 |
| 山地 | +30% | +1 | - | 仅步行单位可进入 |

**设计洞察：**
- **森林**是最核心的战术地形，提供回避加成但消耗移动力
- **堡垒**是战略要点，提供防御+回复双重价值
- **河流/城墙**创造天然阻塞点，形成战场分割
- **移动力消耗**本身就是重要的策略要素

### 2.2 障碍物与通道设计

#### 2.2.1 阻塞点（Choke Point）设计

阻塞点是关卡设计中最重要的空间控制概念，没有高低差时必须依靠障碍物创造。

**阻塞点类型：**

```
┌─────────────────────────────────────────┐
│  类型A：狭窄通道                        │
│                                         │
│  ███████    ███████                     │
│  █  →  █    █  →  █                     │
│  ███████    ███████                     │
│       ↑狭窄通道↑                       │
│                                         │
│  适用场景：控制敌人数量的遭遇战          │
└─────────────────────────────────────────┘

┌─────────────────────────────────────────┐
│  类型B：城门/关卡                       │
│                                         │
│  ███████████████                        │
│  █  门  █  门  █                        │
│  ███████████████                        │
│       ↑可开关的门↑                     │
│                                         │
│  适用场景：攻城战、防守战                │
└─────────────────────────────────────────┘

┌─────────────────────────────────────────┐
│  类型C：桥梁                            │
│                                         │
│  ~~~~~~~~ ████████ ~~~~~~~~             │
│  ~~~~~~~~ █→  → █ ~~~~~~~~             │
│  ~~~~~~~~ ████████ ~~~~~~~~             │
│       ↑河流/悬崖中的唯一通道↑          │
│                                         │
│  适用场景：创造必经的交战点              │
└─────────────────────────────────────────┘
```

#### 2.2.2 可破坏障碍物

增加战场的动态变化：

**炸药桶机制：**
- 被攻击后爆炸，对周围3×3范围造成伤害
- 可以主动引爆用于清场或制造缺口
- 敌方AI会试图远离，可被用于驱赶

**滚石/落石：**
- 触发后沿指定方向滚动
- 对路径上单位造成伤害并击退
- 可以破坏特定墙壁创造新路径

**可破坏墙壁：**
- 耐久度系统，需要多次攻击
- 创造捷径或打破阻塞点
- 提供多路线通关可能

### 2.3 第三方机制

第三方机制指独立于敌我双方的战场元素，是无高低差系统的重要补充。

#### 2.3.1 机关门与开关

```yaml
机关门设计:
  触发方式:
    - 开关: 单位站在指定格子触发
    - 压力板: 任意单位经过时触发
    - 远程触发: 弓箭/法术可触发
  
  门类型:
    - 单向门: 只能从一侧打开
    - 限时门: 开启后N回合关闭
    - 连锁门: 多个开关控制多扇门
    - 伪装门: 看似墙壁，触发后打开
```

**设计要点：**
- 机关门创造分阶段战斗节奏
- 开关位置本身就是战略要点
- 可设计需要"分兵"的关卡

#### 2.3.2 增援点

```yaml
增援机制:
  触发条件:
    - 回合触发: 第X回合开始出现
    - 事件触发: 特定条件满足后
    - 区域触发: 玩家进入某区域后
  
  增援类型:
    - 普通增援: 固定位置，固定频率
    - 奇袭增援: 侧翼/后方出现
    - 无限增援: 直到完成特定目标才停止
  
  设计用途:
    - 制造时间压力
    - 防止龟缩战术
    - 创造多线作战
```

#### 2.3.3 场景互动元素

**火药桶/油桶：**
- 爆炸造成范围伤害和燃烧效果
- 可连锁引爆
- 放置在敌人必经之路形成陷阱

**可推动箱子：**
- 推动到特定位置压住开关
- 堵塞通道创造阻塞点
- 从高处推下造成伤害（如果有高低差则适用）

**治疗/补给点：**
- 占领后每回合回复HP
- 成为双方争夺的焦点

### 2.4 视野与战争迷雾

即使没有高低差，视野限制也能创造丰富的策略深度。

#### 2.4.1 视野机制

```yaml
基础视野:
  标准视野: 5格（所有方向）
  远程单位: 7格
  夜间/洞穴: 3格（需要火把/照明）

视野阻挡:
  森林: 阻挡穿越视野（不能看穿）
  建筑/墙壁: 完全阻挡视野
  高地: 如果存在，视野+2

视野优势:
  先手优势: 发现敌人可提前准备
  射程优势: 远程单位可白嫖
  伏击可能: 隐藏在视野盲区
```

#### 2.4.2 战争迷雾应用

**设计模式：**

1. **伏击关卡**：敌人在迷雾中，需要探路
2. **救援关卡**：被救援对象位置隐藏，需要搜索
3. **撤退关卡**：敌人从迷雾中不断涌出

## 3. 日式战棋关卡设计框架

### 3.1 三维设计模型

```
                    敌人强度
                       │
                       │
         环境大小 ────┼──── 额外机制
                      /│\
                     / │ \
                    /  │  \
              敌人机制  │  第三方机制
                      地形机制
```

**三个维度的取点决定体验偏向：**

| 维度偏向 | 体验特征 | 代表关卡 |
|----------|----------|----------|
| 敌人强度 | 数值验证，考验练度 | BOSSrush关卡 |
| 环境大小 | 空间调度，考验移动 | 大型战场 |
| 额外机制 | 策略验证，考验理解 | 机关解谜关 |

### 3.2 情感曲线设计

```
紧张度
 100% │                                    ╭──────╮
      │                              ╭────╯      │
  85% │                        ╭────╯             │
      │                  ╭────╯                  │
  45% │            ╭────╯                       │
      │      ╭────╯                              │
  20% │──────╯                                   │
      │                                          ╰────────
   0% └──────────────────────────────────────────────────
        开局    展开    高潮    决胜    结束
       (布局)  (试探)  (对拼)  (收割)  (确认)
```

**各阶段设计要点：**

| 阶段 | 紧张度 | 玩家目标 | 设计要点 |
|------|--------|----------|----------|
| 开局 | 20% | 占位布局 | 给玩家思考时间，布置初始阵型 |
| 展开 | 45% | 试探交火 | 首次接敌，测试敌方强度 |
| 高潮 | 85% | 核心对拼 | 主力交锋，技能全开 |
| 决胜 | 95% | 收割残局 | 清理剩余敌人，控制战损 |
| 结束 | 30% | 胜利确认 | 打扫战场，获得奖励 |

### 3.3 敌人类型配置

#### 3.3.1 敌人角色定位

```yaml
输出型:
  特征: 高攻击，低防御
  威胁: 对我方脆皮单位威胁大
  对策: 优先击杀或用坦克护卫

坦克型:
  特征: 高防御，有护卫技能
  威胁: 保护敌方输出，拖延时间
  对策: 绕过或用法术/真实伤害击杀

辅助型:
  特征: 治疗、增益、控制
  威胁: 延长战斗，增加变数
  对策: 优先击杀，打断节奏

突击型:
  特征: 高机动，绕后能力
  威胁: 切后排，打乱阵型
  对策: 保护后排，预留反击单位
```

#### 3.3.2 敌人配置公式

**基础配置比例：**
```
标准遭遇战 = 2输出 + 1坦克 + 1辅助
大型战斗 = 4输出 + 2坦克 + 2辅助 + 1突击
BOSS战 = 1BOSS + 2护卫 + 2杂兵
```

**难度调节：**
- 增加坦克比例 → 战斗拉长，考验续航
- 增加输出比例 → 战斗激烈，考验爆发
- 增加突击比例 → 阵型重要，考验保护

## 4. 实用设计模式

### 4.1 阻塞点设计模式

#### 模式A：城门守卫战

```
┌─────────────────────────────┐
│  敌人援军 →                 │
│                             │
│      ████████████████       │
│   我→█ 敌  敌  门  █        │
│  方  ████████████████       │
│                             │
│  (需要坚守N回合或击杀首领)  │
└─────────────────────────────┘
```

**设计要点：**
- 城门是天然阻塞点
- 敌方援军不断从上方涌入
- 胜利条件：坚守X回合或击杀敌方首领

#### 模式B：桥头阻击战

```
┌─────────────────────────────┐
│  ~~~~  敌方大军  ~~~~       │
│  ~~~~ ████████ ~~~~         │
│  ~~~~ █ 桥 █ ~~~~           │
│  ~~~~ ████████ ~~~~         │
│       我方阵地              │
│       治疗师  弓箭手        │
│       战士    骑士          │
└─────────────────────────────┘
```

**设计要点：**
- 桥梁是唯一的交战点
- 利用地形：弓箭手站位优势
- 治疗师在安全位置提供支援

#### 模式C：分兵合击

```
┌─────────────────────────────┐
│  左路敌军      右路敌军     │
│      │              │       │
│   森林│         河流│       │
│      │              │       │
│  [我方分兵点]               │
│      ↓              ↓       │
│   坦克+治疗     输出+刺客   │
└─────────────────────────────┘
```

**设计要点：**
- 需要分兵同时处理两路
- 左右路地形不同，需要不同配置
- 任何一路失守都可能导致失败

### 4.2 分阶段战斗设计

通过机关门创造分阶段战斗，避免"一波流"的单调感。

```
阶段1：突破外围
┌─────────────────────────────┐
│  外围敌军                   │
│  ████████████████           │
│  █  外围防线    █ ←机关门A  │
│  ████████████████           │
│                             │
│         [开关A] ←需要占领   │
└─────────────────────────────┘

阶段2：攻入内城
┌─────────────────────────────┐
│  内城精英                   │
│  ████████████████           │
│  █  BOSS房间    █ ←机关门B  │
│  ████████████████           │
│                             │
│         [开关B] ←需要占领   │
└─────────────────────────────┘

阶段3：BOSS战
┌─────────────────────────────┐
│        [BOSS]               │
│      精英护卫 精英护卫      │
│                             │
│    (胜利条件：击杀BOSS)     │
└─────────────────────────────┘
```

**设计优势：**
- 每阶段可以休整、补给
- 可以根据阶段调整敌人配置
- 给玩家明确的进度感

### 4.3 多路线设计

提供多条通关路径，增加重玩价值和策略深度。

```
┌─────────────────────────────┐
│        [目标点]             │
│             │               │
│    ┌────────┴────────┐      │
│    │                 │      │
│  森林路            河流路   │
│  (敌人少)          (敌人多) │
│  (有宝藏)          (有捷径) │
│    │                 │      │
│    └────────┬────────┘      │
│             │               │
│        [起点]               │
└─────────────────────────────┘
```

**路线设计原则：**
- 不同路线有不同的风险/收益
- 没有绝对最优解，取决于队伍配置
- 可以设计需要多路线配合的关卡

### 4.4 保护/护送目标设计

无高低差系统中，保护目标需要更精心的地形设计。

#### 保护型关卡

```
┌─────────────────────────────┐
│  敌人进攻方向 → → →         │
│                             │
│  森林  森林  [保护目标]      │
│   ↓     ↓    城墙城墙       │
│  我方防线  我方防线          │
│                             │
│       (坚守N回合)           │
└─────────────────────────────┘
```

**设计要点：**
- 利用森林和城墙创造保护区域
- 保护目标本身可以提供防御加成
- 设计多条敌人进攻路线，防止单一防守

#### 护送型关卡

```
┌─────────────────────────────┐
│  [终点]                     │
│    ↑                        │
│  森林→森林→开阔地           │
│    ↑                        │
│  [护送目标] → 起点          │
│                             │
│  (需要护送目标到达终点)     │
└─────────────────────────────┘
```

**设计要点：**
- 护送目标移动力通常较慢
- 需要清理路线上的敌人
- 可以设计分支路线供玩家选择

## 5. 案例分析

### 5.1 火焰纹章GBA的优秀地图

#### 案例1：烈火之剑 - 草原战役

**地图特点：**
- 中等规模（20×15）
- 大量森林地形分布
- 中央开阔平原作为交战区

**设计亮点：**
1. **森林走廊**：地图两侧森林形成天然通道，适合绕后
2. **中央平原**：无遮蔽的交战区，考验先手和集火
3. **堡垒据点**：地图四角有堡垒，提供战略价值

**战术启示：**
- 森林不仅是防御地形，也是隐蔽接近的路径
- 中央开阔地适合骑兵冲锋，但风险也高
- 堡垒的HP回复效果使其成为必争之地

#### 案例2：封印之剑 - 沙漠关卡

**地图特点：**
- 大面积沙漠地形
- 移动力严重受限
- 隐藏宝物和敌人

**设计亮点：**
1. **移动力惩罚**：沙漠消耗+2移动力，大幅降低机动性
2. **隐藏要素**：沙漠中有隐藏物品和伏兵
3. **路径规划**：存在隐藏的"快速通道"

**战术启示：**
- 移动力惩罚本身就是一种战略深度
- 隐藏要素鼓励探索和侦察
- 飞行单位在沙漠关卡价值凸显

#### 案例3：圣魔之光石 - 防御战

**地图特点：**
- 紧凑的防御地图
- 多个入口需要防守
- 增援从多个方向出现

**设计亮点：**
1. **多路防守**：敌人从3个方向进攻
2. **阻塞点利用**：利用地形狭窄处建立防线
3. **增援压力**：每回合都有新敌人出现

**战术启示：**
- 有限的空间迫使玩家做取舍
- 增援机制防止龟缩战术
- 治疗单位的站位至关重要

### 5.2 天地劫的无高低差设计

天地劫作为国产战棋精品，在无高低差设计上有很多值得学习的地方。

#### 地形互动机制

**元素地形：**
```yaml
火焰地形:
  效果: 进入/经过时受到伤害
  配合: 火系法术可以创造火焰地形
  策略: 用火墙分割战场

霜冻地形:
  效果: 移动力-2，可能触发冰冻
  配合: 冰系法术创造霜冻地形
  策略: 限制敌人机动

雷电地形:
  效果: 概率触发麻痹
  配合: 雷系法术创造雷电地形
  策略: 控制敌人行动
```

**机关互动：**
- 炸药桶可被攻击引爆
- 冰柱可被击碎制造路径
- 开关控制门和升降平台

#### 关卡设计特点

1. **属性循环配合地形**
   - 火克雷，雷克冰，冰克火
   - 地形效果也遵循属性相克
   - 创造"元素战场"的概念

2. **气力技系统**
   - 战斗积累气力
   - 满气力释放强力技能
   - 配合地形可打出combo

3. **五内Build影响战术**
   - 不同的技能选择影响对地形的利用
   - 例如：选择跳跃技能可以越过障碍

### 5.3 梦幻模拟战的地图设计

梦幻模拟战作为经典SRPG，其地图设计强调兵种克制和佣兵系统。

#### 兵种与地形的关系

```yaml
骑兵:
  平原优势: 移动力最大化
  地形弱点: 森林/山地消耗大增
  战术: 绕后突袭，避开复杂地形

飞兵:
  地形免疫: 无视所有地形障碍
  战术价值: 跨越高墙，直取目标
  平衡: 怕被弓箭克制

步兵:
  地形适应: 可进入森林/山地
  战术: 利用地形防御对抗骑兵

弓兵:
  高地优势: 射程+1（如果有高低差）
  射程压制: 控制关键通道
```

#### 佣兵系统的空间意义

- 佣兵跟随英雄行动，增加"单位体积"
- 宽通道才能容纳多个佣兵并排
- 狭窄地形限制了佣兵的展开

## 6. 设计Checklist

### 6.1 基础检查

- [ ] 地图是否有明确的交战区？
- [ ] 是否存在合理的阻塞点？
- [ ] 地形效果是否清晰传达给玩家？
- [ ] 胜利条件是否明确？
- [ ] 失败条件是否合理？

### 6.2 战术深度检查

- [ ] 是否存在多条通关路径？
- [ ] 不同兵种/职业是否有不同的地形适应？
- [ ] 是否有需要利用地形效果才能解开的局面？
- [ ] 机关/互动元素是否增加了战术选择？
- [ ] 敌人配置是否考验玩家的阵型安排？

### 6.3 节奏感检查

- [ ] 关卡是否有清晰的阶段划分？
- [ ] 紧张度曲线是否符合"低→高→低"的节奏？
- [ ] 是否存在可以让玩家喘息的时机？
- [ ] 关卡时长是否适中（建议15-30分钟）？

### 6.4 公平性检查

- [ ] 玩家是否有足够信息做决策？
- [ ] 失败是否可以归咎于可学习的错误？
- [ ] 是否存在"初见杀"的不公平设计？
- [ ] 随机因素是否控制在合理范围？

### 6.5 可访问性检查

- [ ] 色盲玩家是否能区分重要地形？
- [ ] 地形效果是否有清晰的UI提示？
- [ ] 首次出现的机制是否有教学？
- [ ] 难度曲线是否平滑？

## 8. 结语

无高低差的关卡设计并非妥协，而是一种独特的设计哲学。通过精心设计的地形效果系统、巧妙的障碍物布局、丰富的第三方机制，同样可以创造出深度十足的战略体验。

**核心设计原则回顾：**

1. **地形即战略** - 让每块地都有价值
2. **阻塞点创造选择** - 用空间限制引导战术
3. **机制丰富深度** - 机关、增援、互动元素增加变数
4. **节奏决定体验** - 好的关卡像一首曲子，有起伏有高潮

> *"没有高低差，不代表没有高度。"*
> 
> *—— 致每一位SRPG关卡设计师*
